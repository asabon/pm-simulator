# Document Review Skill

本スキルは、PRを発行する前にドキュメント（Markdownファイル）の整合性、表記揺れ、リンク切れを自動チェックするためのガイドラインです。

## 🔍 チェック項目

### 1. ドキュメント間の整合性 (AGENTS.md)
* **ゲームデザインの整合性**:
  * 変更された内容が `docs/game_design.md` や `README.md` の全体方針（通年キャリア型、3大フェーズ、PLアサイン委任、AP制限、人間モデルなど）と矛盾していないか。
* **関連ファイルの更新漏れ**:
  * `game_design.md` が更新された場合、必要に応じて `README.md` の概要・仕様紹介も更新されているか。

### 2. 文章の品質 ＆ 表記ルール
* **誤字脱字・表記揺れ**:
  * 用語（「スプリント」「PM」「PL」「DEV」「隠しパラメータ」等）が表記揺れなく統一されているか。
* **Markdown フォーマット**:
  * 見出しレベル（`#`, `##`, `###`）の階層が正しく崩れていないか。
  * リストやテーブルのマークダウン文法が正常か。

### 3. リンク切れ ＆ ファイル参照
* **相対パス / 内部リンク**:
  * [README.md](file:///e:/work/pm-simulator/README.md) や [game_design.md](file:///e:/work/pm-simulator/docs/game_design.md) への相対パスリンク切れがないか。

## 🛠 実行手順
1. `git diff` で差分のある Markdown ファイル（`.md`）を特定する。
2. 上記のチェック項目に基づき、内容の一貫性とリンクを検証する。
3. 修正が必要な場合はPR発行前にドキュメントを更新する。
